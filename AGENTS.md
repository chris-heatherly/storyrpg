# StoryRPG вЂ” Agent Orientation

Read this file first when working on this codebase.

## What Is This Project?

StoryRPG is an interactive fiction app that blends visual-novel storytelling with tabletop RPG mechanics. Stories are generated by a multi-agent AI pipeline (text, images, audio) and played back as full-screen, image-backed reading experiences where the player makes choices that shape their character's identity. Think "choose your own adventure" with AI-generated prose, illustrations, and optional voice narration.

**Key design principle:** The player never sees stats, dice rolls, or numbers. All mechanics are expressed through narrative prose ("fiction-first").

## Architecture Overview

The system has three execution zones:

1. **Client app** (Expo) вЂ” UI screens, story playback engine, Zustand stores. Runs on web, iOS, and Android.
2. **Proxy server** (Express on Node.js, port 3001) вЂ” Proxies LLM/image/audio API calls (avoids CORS, keeps keys server-side), serves generated story files, manages generation job state.
3. **AI pipeline** (TypeScript, runs in worker process or in-browser) вЂ” Multi-agent story generation using specialized agents, validators, and orchestration pipelines.

Data flow:
```
User triggers generation в†’ Proxy в†’ Worker в†’ AI Pipeline (LLMs, image/audio APIs)
                                                в†“
Validated story JSON + images written to generated-stories/
                                                в†“
Client reads story files в†’ Story Engine в†’ Player experience
```

Choices can have: conditions (flag/stat gates), consequences (flag/stat changes), stat checks (fiction-first resolution), and branching targets.

## Conventions and Patterns

- **Endpoints centralized** вЂ” All URLs in `src/config/endpoints.ts`. Never hardcode URLs elsewhere.
- **Fiction-first** вЂ” No visible stats/numbers in UI. All mechanics expressed narratively.
- **Node polyfills** вЂ” `fs`/`path`/`process` are shimmed via babel and metro config so pipeline code can run in RN/web bundles.
- **Typecheck split** вЂ” Five tsconfig files: `tsconfig.app.json` (narrow app subset), `tsconfig.test.json`, `tsconfig.contracts.json` (shared types for worker payloads), `tsconfig.worker.json`, and base `tsconfig.json`.
- **Tests** вЂ” Co-located `*.test.ts` files, run with Vitest in Node env with RN stubs in `test/stubs/`.
- **Generated content** вЂ” `generated-stories/` contains large JSON + image files and dominates repo size. Runtime artifacts (`.generation-jobs.json`, `pipeline-memories/`, `.ref-images/`) are transient.
- **EXPO_PUBLIC_** prefix вЂ” Required for env vars that need to be accessible in the Expo client bundle.
