# StoryRPG Project Audit

**Date:** 2026-05-28
**Scope:** Whole-project audit ‚Äî architecture, AI pipeline, developer/AI tooling, process/guardrails, goal alignment, and risk. Produced from a 6-dimension read-only audit with independent verification of every headline claim.
**Relationship to other docs:** Complements `docs/TECH_DEBT_AUDIT.md` (2026-05-25), which covers code health well. This doc goes beyond it into product outcomes, tooling, and process. Where the two overlap, the tech-debt code-health claims still hold ‚Äî this report re-sequences around the findings below rather than discarding them.

---

## Executive Read

**The quality machinery has turned against the product goal.** StoryRPG's goal is to ship complete, high-quality, branching, image-backed interactive fiction generated by a multi-agent AI pipeline and played in a public reader app, "fiction-first" (the player never sees stats/dice).

Verified facts:

- **Only 3 of 38** runs in `generated-stories/` contain a `story.json`; **25** contain `99-pipeline-errors.json`. Failures are dominated by hard-gate validators ‚Äî `[TreatmentFidelity]` (22√ó) and `[DramaticStructure]` (17√ó) ‚Äî that abort the entire run with **zero playable output**, discarding completed episodes and full LLM spend.
- `FullStoryPipeline.ts` is **20,937 lines** (grew ~791 since the 2026-05-25 audit named it the #1 file to shrink).
- **No guardrail is enforced** ‚Äî there is no CI and no git hooks ‚Äî while `scripts/auto-commit-push.sh` blind-pushes `git add -A` to `main`.
- The proxy is reachable on the public internet via an ngrok tunnel with wide-open CORS and unauthenticated mutating routes.

---

## Cross-Cutting Themes

1. **Guardrails by convention, not tooling** ‚Äî `validate`, `lint`, `check:reader-boundary`, `typecheck:*` all exist but nothing runs them automatically.
2. **Scaffolding mistaken for progress** ‚Äî phase extraction is 2/12 done with `WorldBuildingPhase` scaffolded-but-unwired; dead code (`SeasonValidator`, `EpisodePipeline`) is counted as live.
3. **Quality is over-gated AND self-graded** ‚Äî a few structural gates hard-abort runs while most validators only warn; QA scores are LLM self-assessment that defaults to 50 on parse failure.
4. **No measurement loop** ‚Äî the 0-100 quality score is computed-and-discarded; the 25/38 collapse is invisible outside the filesystem.
5. **Cost/latency levers unused** ‚Äî no prompt caching, no model tiering, unbounded multiplicative retry fan-out.
6. **Two untyped monoliths** concentrate risk: `FullStoryPipeline.ts` (~950KB, `@ts-nocheck`), `imageGenerationService.ts` (6,561 lines).
7. **Knowledge fragmentation** ‚Äî 3 divergent skill ecosystems; the best are gitignored, the committed ones are thinnest, zero target Claude Code.
8. **Fragile secret/safety surface** ‚Äî publicly-reachable proxy, `EXPO_PUBLIC_*` provider keys, ~13MB of job-state backups in git history.

## Landmines (fix first ‚Äî secret leak / data loss / main-branch corruption)

### L1. Lock down the publicly-reachable proxy `[risk, architecture]`
- **Evidence:** `proxy-server.js:126` ‚Üí `app.use(cors({ origin: true, credentials: true }))` reflects any origin with credentials; `PROXY_PUBLIC_URL=https://‚Ä¶ngrok-free.dev` means it's internet-reachable when the tunnel is up; mutating routes (project/job/art `DELETE`, file writes) lack an auth gate and several fall back to `process.env` provider keys.
- **Fix:** Replace `origin:true` with an explicit allowlist; require auth (the `proxy/authRoutes.js` / Passport stack already exists) on all mutating + key-using routes; default the ngrok tunnel off.

### L2. Stop the auto-committer + purge leaked artifacts `[process, risk]`
- **Evidence:** `scripts/auto-commit-push.sh` runs `git add -A` ‚Üí `git push origin main` with no validation and a stale `REPO_DIR=/Users/chrisheatherly/StoryRPG_New`. Tracked artifacts: `.generation-jobs.json.backup-*` (6.5MB √ó2), `.worker-jobs.json.backup-*` (447KB), `.model-cache.json`. `.gitignore` ignores the live files but not `*.backup-*` or `.model-cache.json`.
- **Fix:** Retire/disable the script (or gate behind `validate`, push to a wip branch, never `main`); `git rm --cached` the artifacts; add `*.backup-*`, `.model-cache.json`, `.generator-settings.json`, `.codex-expo-web.*` to `.gitignore`; extend `clean:runtime` to cover them.

### L3. Close the reader secret-leak path `[risk, process]`
- **Evidence:** `.env` has real `EXPO_PUBLIC_GEMINI_API_KEY` / `EXPO_PUBLIC_OPENAI_API_KEY`; Expo inlines `EXPO_PUBLIC_*` into the client bundle. `scripts/check-reader-boundary.mjs` only greps import paths + a *name* (`ANTHROPIC_API_KEY`) and its `dist-reader` value-scan runs only `if (fs.existsSync(distReader))`; `validate:reader` never builds `dist-reader`, so the value scan never executes. Safe today only because the reader graph happens not to import `config.ts`.
- **Fix:** Move provider keys server-side (proxy) off the `EXPO_PUBLIC_` prefix; make the boundary script run `reader:export` first, fail if `dist-reader` is absent, and scan the bundle for key *values* + `AIza[0-9A-Za-z_-]{35}` patterns.

### L4. Stop silent content/progress loss `[ai-pipeline, reader]`
- **Evidence:** `BaseAgent.handleTruncation` truncates at the last complete object and can drop scenes/beats while reporting success; `gameStore.ts:333-337` swallows persistence-write failures with `console.warn`.
- **Fix:** Downgrade truncation recovery to a gate-able logged warning visible to the final-contract validator; surface persistence-write failures to the user.

## Track A ‚Äî Enforce guardrails (quick wins)

- **A1. Add blocking CI** ‚Äî `.github/workflows/ci.yml` on PR+push: `npm ci && typecheck && lint && test && check:reader-boundary`, required-to-merge.
- **A2. Make lint able to fail** ‚Äî run lint with `--max-warnings <current>` as a ratchet (currently 431 warnings / 0 errors); promote `no-restricted-imports` (endpoints contract, ~0 violations) to `error`; set `ban-ts-comment: warn` with an allowlist of existing `@ts-nocheck` files.
- **A3. Monolith line-count ratchet** ‚Äî CI check that fails if `FullStoryPipeline.ts` / `imageGenerationService.ts` grow.
- **A4. Pre-commit hook** ‚Äî husky + lint-staged: eslint on staged files + a staged-path guard blocking runtime-artifact globs.
- **A5. Wire existing-but-unrun gates** ‚Äî run `test:coverage` in CI to enforce the thresholds in `vitest.config.ts`; add a PR/nightly `test:e2e` (Playwright story-playthrough) job.

## Track B ‚Äî Recover the product (make stories ship again)

- **B1. Tier the validators** ‚Äî In `StoryArchitect.ts:1751-2152`, split hard correctness gates (broken links, orphan scenes, unauthored outcome tiers) from craft/fidelity advisories (`TreatmentFidelity`, `DramaticStructure`, `ThemePressure`) that, after retries, degrade to recorded warnings on a still-written `story.json`. Document the blocking-vs-advisory matrix in `STORY_QUALITY_CONTRACT.md`.
- **B2. Never discard generated work** ‚Äî On any abort, write a partial `story.json` + failure report so completed episodes survive a later-episode failure.
- **B3. Track quality + success rate over time** ‚Äî Append `overallScore` + sub-scores + run outcome to an append-only JSONL ledger per run (the score in `IntegratedBestPracticesValidator.ts:745` is currently computed-and-discarded); render a trend; define ship/warn/block thresholds.
- **B4. Validator registry** ‚Äî One declarative manifest (stage ‚Üí validators ‚Üí blocking|advisory) consumed by the 3‚Äì4 current dispatch sites; fix the stale "not yet integrated" comment in `validators/index.ts:153`; delete or wire dead `SeasonValidator`; add tests for the ~15 untested wired validators.

## Track C ‚Äî Cost & correctness of the pipeline

- **C1. Anthropic prompt caching** ‚Äî `cache_control:{type:'ephemeral'}` on the static system prefix in `BaseAgent.ts`; stable-prefix-first message ordering.
- **C2. Per-agent model tiering** ‚Äî planning = Opus/Sonnet; mechanical + QA grading = Haiku. `AgentConfig` already supports per-agent model.
- **C3. Decorrelate QA grading** ‚Äî different model as judge than author; failed-parse QA score fails closed instead of defaulting to 50; golden-story eval set to calibrate.
- **C4. Per-story token/cost budget** ‚Äî consume the existing `LlmCallObservation` usage stream to abort runaway stories.

## Track D ‚Äî Skills, agents & workflow tooling

The best skills already exist but target the wrong tools / are uncommitted. Mostly consolidate-adapt-commit.

- **D1. One committed source of truth for skills** ‚Äî promote/un-ignore the gold `.cursor/skills`; add `scripts/sync-skills.mjs`.
- **D2. Author Claude Code skills** under `.claude/skills/`: `pipeline-debugging`, `pipeline-validation`, `reader-generator-safety` (net-new, safety-critical), `proxy-server`, `story-playback-reader`, `image-generation-team`, `testing-tooling`, `story-structure-rules`.
- **D3. Subagents** ‚Äî reader-boundary auditor; validator-impact analyzer.
- **D4. Slash-commands** ‚Äî `/generate-story`, `/check-debt`, `/verify-reader`.
- **D5. Refresh orientation docs** ‚Äî update `AGENTS.md` (reader/generator split, `STORYRPG_APP_TARGET`, ports, auth subsystem, DB layer, `check:reader-boundary`); add a thin `CLAUDE.md`; add `Active | Archived | Superseded` banners and move completed one-off plans to `docs/archive/`.
